Quote Reflections

Programming Design Systems: Introduction

“We are now in transition from an object-oriented to a systems-oriented culture. Here change emanates, not from things, but from the way things are done.”

It connects to how my work has expanded from making static images to building interactive systems.

“Digital products allow users to interact with their content … a linear narrative is replaced by a set of complex states and transitions.”

It reminds me that digital design is not just about visuals. Time, motion, and changes become part of the design. It makes me think more about behavior, not only form.

“What happens when we try to redefine the graphic design curriculum using a programming language as the tool for the designer?”

Seeing code as a design tool changes how I make things. Using TouchDesigner, I can test ideas and see what happens right away — it helps me think and design at the same time.

What Do Prototypes Prototype? (Houde & Hill, 1997)

“Prototypes are any representation of a design idea, regardless of medium.”

This shows that a prototype can be anything — a sketch, a test, or even a rough idea. It makes me feel freer to try different things instead of making one perfect model.

“Role, look and feel, and implementation are distinct dimensions of prototypes.”

It helps me see that each prototype can focus on one thing — how it works, how it feels, or how it’s built. I can test each part separately and understand my design better.

“No single prototype can answer all questions about an interactive artifact.”

It reminds me that one prototype doesn’t have to do everything. Making different versions helps me learn step by step instead of worrying about perfection.
